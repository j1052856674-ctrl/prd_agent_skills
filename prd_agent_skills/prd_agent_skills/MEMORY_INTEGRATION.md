# 记忆系统集成说明

## 🧠 记忆系统 vs 知识库

### 1. 双重存储机制

#### 记忆系统（.claude/memory/）
- **用途**：Claude Code的长期记忆
- **存储位置**：`C:\Users\fanjiang\.claude\projects\...\memory\`
- **内容**：用户偏好、模板特征、历史决策
- **访问**：自动在对话中加载，无需手动调用
- **特点**：持久化，跨会话保持

#### 知识库（personal_knowledge_base/）
- **用途**：技能运行时数据和工具
- **存储位置**：`prd_generator/personal_knowledge_base/`
- **内容**：样本、模板、AI模型、工具脚本
- **访问**：通过skill代码调用
- **特点**：技能功能的一部分，实时可用

### 2. 同步机制

#### 记忆 → 知识库
- 用户偏好更新到user_profile/profile.json
- 模板特征同步到样本库
- 使用统计数据更新索引文件

#### 知识库 → 记忆
- 新增样本记录到记忆
- 用户行为影响记忆更新
- 质量评估反馈到偏好

## 🔧 核心功能联动

### 1. 模板推荐
- 记忆：记住用户喜欢"技术型详细风格"
- 知识库：提供data_intelligence_agent模板
- 结果：推荐符合偏好的模板

### 2. 架构图生成
- 记忆：记住"四阶段架构"偏好
- 知识库：提供四阶段样本和配色
- 结果：生成一致的架构图

### 3. 质量评估
- 知识库：生成内容质量评估
- 记忆：更新用户偏好权重
- 结果：后续推荐更精准

## 📊 使用流程

```
用户输入
    ↓
记忆系统：加载用户偏好
    ↓
知识库：提供样本和工具
    ↓
Skill：处理生成内容
    ↓
双重更新：记忆 + 知识库
    ↓
下次：更精准的推荐
```

## 🎯 优势

1. **双重保障**：记忆确保跨会话一致性，知识库确保实时可用性
2. **智能进化**：用户行为持续优化推荐
3. **灵活扩展**：新增样本自动更新记忆
4. **性能优化**：记忆快速加载，知识库丰富存储

## 📝 维护建议

1. **定期同步**：新增重要模板时更新记忆
2. **监控质量**：跟踪生成质量评估结果
3. **用户反馈**：及时记录用户偏好变化
4. **样本更新**：高质量生成内容沉淀到知识库
