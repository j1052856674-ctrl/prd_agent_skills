二、需求目标

  • 用户输入表名+需求描述 → 验证表名 → 需求澄清 → 生成SQL
  • 用户选择表开发 → 上传Excel+描述 → 需求澄清 → 生成DDL
  • SQL执行错误反馈机制
  • SQL模板库（异步沉淀）
  • 安全拦截机制
  • 异常自愈库（Case Base）

4.4 安全拦截机制

安全拦截库（Safety Guard）
  • SQL注入检测 - 检测恶意SQL关键字
  • 越权字段检测 - 禁止访问敏感字段
  • 隐私保护规则 - 自动脱敏敏感数据
  • 大查询限制 - 限制全表扫描
  • 危险操作拦截 - DROP/DELETE/TRUNCATE

4.5 SQL模板库（异步沉淀）

SQL模板库（Template Base）
  • 模板分类 - 聚合、排序、窗口函数、JOIN等
  • 使用场景 - 生成SQL时匹配相似模板
  • 沉淀方式 - 执行成功后异步入库
  • 模板字段 - 模板名称、SQL模板、适用场景、创建时间

4.6 异常自愈库（Case Base）

异常自愈库（Case Base）
  • 记录内容 - 错误信息 + 修复方案
  • 使用方式 - SQL生成失败时查询自愈库
  • 自愈流程 - 匹配错误 → 应用修复 → 重试(1次)
  • 成功案例沉淀 - 自愈成功后入库
